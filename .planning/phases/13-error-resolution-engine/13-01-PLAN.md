# Phase 13 Plan 01: Error Logging Infrastructure

**Structured error capture from all TUI pages with categorized storage**

## Objective
Create error module with structured logging, wire capture into existing error paths across Welcome, Setup, and Bridge pages.

## Tasks

### Task 1: Create error module structure
- `type: auto`
- New src/error/mod.rs — ErrorEntry, ErrorCategory (Welcome/Setup/Bridge), ErrorSeverity (Warning/Error/Critical)
- New src/error/logging.rs — File-based ErrorStore at ~/.interclaude/errors/, JSON-lines format

### Task 2: Wire ErrorStore into App
- `type: auto`
- Add error_store to App, initialize on start
- Detect error-like messages in push_setup_log/push_bridge_log

### Task 3: Capture errors from all pages
- `type: auto`
- Welcome: missing required deps
- Setup: SSH test failures, activate failures
- Bridge: connection failures, transport errors

## Success Criteria
- [ ] Errors captured to ~/.interclaude/errors/ as JSON-lines
- [ ] Build succeeds
