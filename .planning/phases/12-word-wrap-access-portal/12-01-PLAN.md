# Phase 12 Plan 01: Universal Word Wrapping

**Ensure all text content in all TUI tabs wraps correctly within container boundaries**

## Objective
Audit every rendering widget across all 4 TUI pages and ensure word wrapping is applied consistently. Convert message lists from single-line List items to wrapped multi-line rendering.

## Context
- 11 existing `Wrap` usages across welcome.rs, setup.rs, bridge.rs
- Bridge message lists (outbox/inbox) use ratatui `List` + `ListItem` — single-line, no wrapping
- Some widgets may truncate on narrow terminals (< 60 cols)
- ratatui Paragraph supports `.wrap(Wrap { trim: true/false })`
- ListItem can contain multi-line `Text` for wrapped content

## Tasks

### Task 1: Audit all rendering widgets
- `type: auto`
- Read every `draw_*` function in welcome.rs, setup.rs, bridge.rs, status_bar.rs
- Identify any Paragraph without `Wrap`
- Identify any List with content that could exceed panel width
- Identify any raw Span that could overflow its container
- Document gaps

### Task 2: Convert message lists to wrapped rendering
- `type: auto`
- In bridge.rs `draw_messages()`, the outbox and inbox use `List` widget with `ListItem` items
- Update ListItem construction to use `Text::from(lines)` where content is pre-wrapped to available width using `textwrap` logic or ratatui's built-in line wrapping
- Ensure "You →" / "Remote →" prefixes render correctly with wrapped continuation lines
- Maintain scroll behavior (outbox_scroll, inbox_scroll) with wrapped items

### Task 3: Add Wrap to remaining unwrapped Paragraphs
- `type: auto`
- Add `.wrap(Wrap { trim: true })` to any Paragraph widgets missing it
- Check pipeline labels, status text, nav bars, help overlay text
- Verify rendering at 40-col terminal width doesn't overflow

## Verification
- Resize terminal to 40 columns wide
- Navigate all pages: Welcome, Setup, Bridge
- Verify no text overflows panel boundaries
- Verify long messages in bridge wrap across multiple lines
- `cargo build` succeeds

## Success Criteria
- [ ] All Paragraph widgets have Wrap applied
- [ ] Bridge message lists wrap long content
- [ ] No text overflow visible at 40-col terminal width
- [ ] Build succeeds with no new warnings
