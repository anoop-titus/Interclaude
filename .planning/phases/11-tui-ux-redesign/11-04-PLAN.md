**UI/UX Review Findings (Global):**
- Reconnection attempts not shown — users see "Connected" flip to nothing
- Esc means "back" on Setup/Bridge but "quit" on Welcome — inconsistent
- No Ctrl+C handler (users will instinctively try it)
- Status bar uses * and x ASCII — could use Unicode on capable terminals
- On narrow terminals, "Up"/"Dn"/"On"/"Off" is cryptic
- Ctrl+L means "Launch Slave" on Bridge but nothing elsewhere

**Prior decisions:**
- BridgeEvent::ConnectionStatus(String) carries status text
- autossh handles reconnection in background
- Connection status stored as String in app.connection_status
</context>

<task type="auto">
  <name>Task 1: Enhance status bar with reconnection states and Unicode indicators</name>
  <files>src/tui/status_bar.rs, src/app.rs, src/bridge/engine.rs</files>
  <action>
    Add richer connection states to the status bar:

    Replace the simple string-based connection status with an enum in app.rs:
    ```
    ConnectionState { Disconnected, Connecting, Connected(Role), Reconnecting(u32), Failed(String) }
    ```
    where Reconnecting(u32) tracks the attempt number.

    In engine.rs, when autossh or transport health checks detect a drop, emit BridgeEvent::ConnectionStatus with "Reconnecting (attempt N)..." text. When health recovers, emit "Connected" again.

    In status_bar.rs:
    - Replace ASCII `*` with Unicode `●` for connected (green)
    - Replace ASCII `x` with Unicode `○` for disconnected (red)
    - Add `◌` for reconnecting (yellow, pulsing via frame counter)
    - On narrow terminals, replace "Up"/"Dn" with "●"/"○" (the dots are more universal than text)
    - Show reconnection attempt count: "Reconnecting ●●○" (filled dots = attempts)

    Add a session duration timer: when Connected, show elapsed time since connection in the status bar (HH:MM format). Add `connected_at: Option<std::time::Instant>` to App, set it when connection confirmed.
  </action>
  <verify>cargo build compiles. Run binary — confirm Unicode dots in status bar. Check narrow terminal rendering (resize terminal to 40 cols).</verify>
  <done>Status bar uses Unicode indicators. Reconnection state visible with attempt count. Session duration shown when connected. Narrow terminals render cleanly.</done>
</task>

<task type="auto">
  <name>Task 2: Fix key binding consistency and add Ctrl+C handler</name>
  <files>src/tui/mod.rs, src/app.rs</files>
  <action>
    Key binding fixes:

    1. Add Ctrl+C handler: In handle_input(), add KeyCode::Char('c') with CONTROL modifier → set app.running = false (same as Ctrl+Q). This is what every terminal user expects.

    2. Make Esc consistent: On Welcome page, Esc currently quits. Change it to do nothing on Welcome (or show a "Press Ctrl+Q to quit" hint). Esc should always mean "go back" when there's a page to go back to, and be a no-op on the first page. Ctrl+Q and Ctrl+C are the quit keys.

    3. Remove the Esc-to-quit on Welcome from the nav bar. Replace with: "[Enter] Continue  [Ctrl+Q] Quit"

    4. On Bridge page, add Ctrl+H as a help overlay toggle: when pressed, show a semi-transparent overlay listing all keybindings for the current page. Add `show_help_overlay: bool` to App. Render as a centered Paragraph widget with Clear underneath. Press Ctrl+H or Esc to dismiss.

    5. Ensure all pages show consistent Ctrl+Q in their nav bars (it's the universal quit).
  </action>
  <verify>cargo build compiles. Run binary — press Ctrl+C on any page, confirms quit. Press Esc on Welcome — should NOT quit. Press Ctrl+H on Bridge — help overlay appears.</verify>
  <done>Ctrl+C quits from any page. Esc consistently means "back" (no-op on Welcome). Help overlay available via Ctrl+H on Bridge. All nav bars show Ctrl+Q for quit.</done>
</task>
