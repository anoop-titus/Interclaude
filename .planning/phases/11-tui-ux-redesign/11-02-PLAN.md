**UI/UX Review Findings (Setup Page):**
- 10 fields all visible at once — overwhelming for first-time users
- No visual grouping between Connection fields and Transport/Redis fields
- Redis password shown as plaintext in form
- No input validation feedback (invalid IP/port discovered only on test)
- Key bindings are dense (C-T, C-G, C-S, C-P, C-D, C-A) — hard to remember
- Activate button right-aligned and easy to miss
- Tab/Down both advance, but Enter only cycles selectors — inconsistent

**Prior decisions:**
- Settings saved to ~/.interclaude/config.toml (TOML serialization)
- ConnectionKind enum with cycle()
- TransportKind enum with Rsync/Mcp/Redis variants
</context>

<task type="auto">
  <name>Task 1: Group form fields with visual separators and progressive disclosure</name>
  <files>src/tui/setup.rs, src/app.rs</files>
  <action>
    Restructure the form into 3 visual groups with section headers:

    Group 1 — "Connection" (always visible):
      Remote Host, Connection Type, User, Port, Key Path, Remote Dir

    Group 2 — "Transport" (always visible):
      Transport selector (rsync/MCP/Redis)

    Group 3 — "Redis Configuration" (visible ONLY when Transport = Redis):
      Redis Host, Redis Port, Redis Password

    Add section header lines rendered as bold colored spans with a horizontal rule character (─).
    Example: "── Connection ──────────" in Cyan, "── Transport ──────────" in Magenta.

    Add a `show_redis_config()` method to App that returns true only when settings.active_transport == TransportKind::Redis.

    When Redis group is hidden, Tab/Down should skip from Transport directly back to RemoteHost (wrap around). Update SetupField::next() and prev() accordingly — add a method that takes the current transport to determine skip behavior, or add the skip logic in the input handler.

    Move the Activate button to be more prominent: center it below the form, full-width with a clear visual border, not right-aligned.
  </action>
  <verify>cargo build compiles. Run binary, navigate to Setup — confirm 3 groups visible. Switch transport away from Redis — Redis fields disappear. Switch back — they reappear.</verify>
  <done>Form fields grouped with section headers. Redis config hidden when not using Redis transport. Activate button centered and prominent. Tab navigation skips hidden fields.</done>
</task>

<task type="auto">
  <name>Task 2: Mask Redis password and add inline validation hints</name>
  <files>src/tui/setup.rs, src/app.rs, src/config/settings.rs</files>
  <action>
    Password masking: When rendering the RedisPassword field, display '*' characters instead of the actual value. The actual value remains in settings.redis.password — only the display changes. When the field is focused, show the masked length plus a blinking cursor.

    Inline validation hints for text fields:
    - RemoteHost: If non-empty and doesn't match a basic hostname/IP pattern (contains spaces, starts with /, etc.), show a yellow "?" indicator after the field value
    - SshPort: If non-empty and not a valid port number (1-65535), show yellow "?" indicator
    - RedisPort: Same as SshPort
    - KeyPath: If non-empty and expanded path doesn't exist on disk, show yellow "?" indicator (use std::path::Path::new().exists() — this is sync but fast for local filesystem)

    Render validation indicators as a colored span after the field value: green checkmark-like "ok" when valid, yellow "?" when suspicious. Only show indicators for non-empty fields.

    Simplify nav bar: Remove individual key labels for C-T, C-G, C-P, C-D. Keep only: [Tab] Navigate, [C-A] Activate, [C-S] Save, [Esc] Back. The other shortcuts still work but don't need nav bar space — they're discoverable via the tutorial panel.
  </action>
  <verify>cargo build compiles. Run binary — Redis password shows as asterisks. Type invalid port "99999" — yellow indicator appears. Type valid host — green indicator appears.</verify>
  <done>Redis password masked with asterisks. Inline validation shows indicators for host, ports, key path. Nav bar simplified to essential shortcuts only.</done>
</task>
