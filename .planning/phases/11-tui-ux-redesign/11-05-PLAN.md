**UI/UX Review Findings (Bridge Advanced):**
- Status bar (health + pipeline) takes significant vertical space — ~8 rows on 24-row terminal, leaving only ~8 for messages
- No auto-transport recommendation — users must guess which transport is best
- Bridge log hidden — only shows last entry
- Submit button is non-interactive

**Prior decisions:**
- TransportSelector manages all 3 transports, tracks health[3]
- BridgeEngine does health checks every 15s
- transport_health: [bool; 3] in App
- BridgeEvent::HealthUpdate(kind, healthy) sent from engine
</context>

<task type="auto">
  <name>Task 2: Auto-recommend best transport based on health and latency</name>
  <files>src/bridge/engine.rs, src/tui/bridge.rs, src/app.rs</files>
  <action>
    Add transport recommendation logic to BridgeEngine:

    Create a `recommend_transport()` method that evaluates:
    - Priority order when all healthy: Redis (fastest, <0.1s) > MCP (~0.5s) > rsync (2-5s)
    - If current transport is unhealthy, suggest switching to the best healthy alternative
    - If no transport is healthy, suggest rsync (most reliable fallback)

    Emit a new BridgeEvent::TransportRecommendation(TransportKind, String) where String is the reason (e.g., "Redis has lowest latency").

    In App, add `transport_recommendation: Option<(TransportKind, String)>`.

    In bridge.rs, show the recommendation in the transport header:
    - If current transport matches recommendation: no extra indicator
    - If recommendation differs from current: show a subtle hint next to the transport selector: "Suggested: Redis (lowest latency)" in yellow
    - The user can press the corresponding number key to switch, or ignore

    Don't auto-switch — just recommend. The user decides. This respects the "all 3 transports user-selectable" design decision.

    Trigger recommendation recalculation on every HealthUpdate event.
  </action>
  <verify>cargo build compiles. Verify recommendation logic: when Redis is healthy and user is on rsync, a "Suggested: Redis" hint should appear in the header.</verify>
  <done>Transport recommendation calculated from health checks. Subtle suggestion shown in header when a better transport is available. No auto-switching — user decides.</done>
</task>
