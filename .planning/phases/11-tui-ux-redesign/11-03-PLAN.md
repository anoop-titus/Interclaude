**UI/UX Review Findings (Bridge Page):**
- Input bar doesn't distinguish command vs message — mental model unclear
- Message list has no scrolling — older messages disappear
- Delivery pipeline uses symbols (>>, (), .., OK, <<, <-, ++) requiring learning
- Empty states give no context about what to type
- Status bar takes significant vertical space on small terminals
- Bridge log hidden inside status panel — only shows last entry

**Prior decisions:**
- Messages stored as Vec<MessageEntry> in App
- MessageDirection enum: Outbound/Inbound
- DeliveryStatus with 10 variants and label()/symbol() methods
- compose_input: String for text input
</context>

<task type="auto">
  <name>Task 2: Simplify delivery pipeline to human-readable labels</name>
  <files>src/tui/bridge.rs, src/app.rs</files>
  <action>
    Replace the symbol-based pipeline with word-based labels:

    Update render_delivery_pipeline() to use clear words instead of symbols:
    - Old: [>>]SENT->[()]READ->[..]EXEC->[OK]DONE->[<<]RPLY->[-<]RECV->[++]GOT
    - New: SENT → READ → RUNNING → DONE → REPLYING → RECEIVING → COMPLETE

    Use a cleaner visual style:
    - Completed stages: green text, bold
    - Current stage: cyan text, bold, with a pulsing indicator (use ● character)
    - Future stages: dark gray text
    - Arrows between stages: light gray "→"

    Also update DeliveryStatus::label() in app.rs to use these friendlier names:
    - Executing → "RUNNING"
    - Executed → "DONE"
    - ReceivingReply → "RECEIVING"
    - ReceivedReply → "COMPLETE"
    - (Keep Delivered, Read, Replying as-is — they're already clear)

    Update DeliveryStatus::symbol() to use single Unicode characters:
    - Delivered: "→", Read: "◉", Executing: "⟳", Executed: "✓", Replying: "←", ReceivingReply: "⇐", ReceivedReply: "✔"
    - DeliveryFailed: "✗", ExecutionError: "!", Timeout: "⏱"

    In the message list, show the symbol next to each message instead of the old two-char codes. This is more compact and internationally readable.
  </action>
  <verify>cargo build compiles. Run binary — confirm pipeline shows word labels. Check message list shows Unicode status symbols.</verify>
  <done>Pipeline uses human-readable words. Status symbols are Unicode single characters. Current stage visually highlighted with pulsing indicator.</done>
</task>
