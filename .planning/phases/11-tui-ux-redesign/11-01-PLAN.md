**UI/UX Review Findings (Welcome Page):**
- Deps don't distinguish required vs optional — user doesn't know what blocks them
- No loading spinner during async dep checks
- No auto-advance when all required deps present
- Version strings from some tools are noisy (multi-line stderr)
- The page is a gate with no indication of consequences of missing optional deps

**Prior decisions:**
- ratatui 0.29 patterns established across all TUI files
- crossterm 0.28 for terminal events
- Async dep checks already use tokio::process::Command
</context>

<tasks>

<task type="auto">
  <name>Task 1: Split dependencies into Required and Optional groups with loading state</name>
  <files>src/app.rs, src/tui/welcome.rs, src/tui/mod.rs</files>
  <action>
    Modify DepCheck struct in app.rs to include a `required: bool` field.

    In mod.rs check_dependencies(), mark these as required: ssh, rsync, claude. Mark these as optional: mosh, autossh, fswatch, redis-cli. Add a `dep_check_complete: bool` field to App.

    In welcome.rs, split the dependency list into two visual sections:
    - "Required" section with green/red indicators
    - "Optional" section with green/yellow indicators (yellow = missing but non-blocking)

    Add a simple loading indicator: while dep_check_complete is false, show "Checking system dependencies..." with a rotating spinner character (use frame counter mod 4 for |/-\ cycle). The 100ms poll loop in mod.rs already provides the tick.

    Clean up version display: truncate to first 40 chars, strip any line that starts with "Usage:" or "Warning:".
  </action>
  <verify>cargo build compiles without new errors. Run the binary and confirm Welcome page shows two groups.</verify>
  <done>Dependencies split into Required/Optional sections. Loading spinner visible during checks. Version strings clean.</done>
</task>

<task type="auto">
  <name>Task 2: Auto-advance when all required deps are present</name>
  <files>src/tui/welcome.rs, src/tui/mod.rs, src/app.rs</files>
  <action>
    Add an `all_required_met()` method to App that returns true when all required deps are available.

    After dep checks complete (dep_check_complete = true), if all_required_met():
    - Show a green banner: "All required dependencies found! Press Enter to continue..."
    - After 2 seconds (20 poll ticks), auto-advance to Setup page
    - User can still press Enter immediately to skip the wait

    If NOT all_required_met():
    - Show a red banner listing missing required deps with install hints
    - Show "Press Enter to continue anyway (some features may not work)"
    - Do NOT auto-advance

    Update the nav bar to reflect state: when auto-advancing, show countdown "[Enter] Continue (auto in Xs)"
  </action>
  <verify>cargo build compiles. Test with all deps present — should auto-advance after 2s. Test concept: if ssh were missing, it should show red banner.</verify>
  <done>Auto-advance works when all required deps present. Missing required deps show clear install instructions. User can always manually advance.</done>
</task>
